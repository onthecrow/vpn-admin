# VPNAdmin

Compose Multiplatform tool for managing the `configs` Firestore collection of the **OnthecrowVpn** Firebase project. Built for macOS (Desktop) first, Android also supported.

## Running

- Desktop: `./gradlew :desktopApp:run` (hot reload: `./gradlew :desktopApp:hotRun --auto`)
- Android: `./gradlew :androidApp:installDebug` then launch from device/emulator

Both entry points block on Firebase init + admin sign-in before showing the UI.

## One-time Firebase setup

In the [Firebase Console](https://console.firebase.google.com/) → **OnthecrowVpn** project:

1. **Register an Android app**
   Project Settings → *Your apps* → Add app → Android. Package name `com.onthecrow.vpnadmin`. Download `google-services.json` → place it at `androidApp/google-services.json` (gitignored).
2. **Register a Web app**
   *Your apps* → Add app → Web. Needed because the GitLive Firebase SDK on JVM/Desktop initializes manually from a Web-style config (no `google-services.json` parsing on desktop). Copy the `firebaseConfig` values into `desktopApp/firebase-admin.properties`.
3. **Enable Authentication → Sign-in method → Email/Password**.
4. **Authentication → Users → Add user** — create a single admin (e.g. `admin@onthecrow.tech` + strong password). The app uses these credentials silently at startup.
5. **Firestore → Rules** — restrict access to that admin:
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /configs/{document=**} {
         allow read, write: if request.auth != null
           && request.auth.token.email == "admin@onthecrow.tech";
       }
     }
   }
   ```

## Local files you must create (gitignored)

| File | What it holds |
|---|---|
| `androidApp/google-services.json` | Generated by Firebase Console step 1 |
| `androidApp/src/main/assets/firebase-admin.properties` | Just `admin.email` and `admin.password` — see `firebase-admin.properties.template` next to it |
| `desktopApp/firebase-admin.properties` | All Firebase config (apiKey, projectId, applicationId, storageBucket, gcmSenderId, authDomain) + `admin.email` / `admin.password` — see `firebase-admin.properties.template` |

`*.template` files are committed as references; copy them without the suffix and fill in real values. The `.gitignore` already excludes the real ones plus any `service-account.json`.

For desktop you can also override the lookup with `-Dfirebase.config=/abs/path/to/firebase-admin.properties`.

## Source layout

- `shared/src/commonMain` — models (`data/`), repository (`data/FirestoreConfigRepository`), Compose screens (`ui/screens/`), Compose Nav3 routes (`nav/`), Firebase `expect` (`firebase/`)
- `shared/src/androidMain` — Firebase init using auto-bootstrapped Android SDK; long-press context menu
- `shared/src/jvmMain` — Firebase init via GitLive Java SDK shim + manual `FirebaseOptions`; right-click context menu
- `androidApp` — single Activity that bootstraps the application context, then `App()`
- `desktopApp` — `application { Window { App() } }`
